# HyperFrames 宣传视频提示词

用途：生成一个竖屏短视频，用来介绍 ShuttlePoseReview。视频不需要配音，只需要画面、字幕和节奏，后续可以在剪辑软件里自行加入人声。

## 基本规格

- 画幅：竖屏 1080x1920。
- 时长：60-75 秒。
- 音频：不需要配音，可以保留轻微背景音乐，也可以完全静音。
- 字幕：使用 `docs/promo_subtitles_zh.srt`。
- 风格：竖屏知识分享 + 开源项目演示。不要模仿任何特定账号或创作者的脸、声音、口癖和个人包装，只参考“一个人认真介绍项目 + 旁边展示项目画面”的信息结构。
- 人物呈现：不要真人出镜。用项目作者 GitHub 头像作为讲解者头像，可以做轻微表情、点头、侧身看屏幕等动画，但不要伪造成真实自拍视频。
- 视觉主题：深绿色羽毛球场、白色球场线、荧光绿色重点标注、手机 UI 屏录、骨架姿态点线。
- 边界声明：强调这是训练复盘和开源研究工具，不承诺医疗、专业教练或竞技判断。

## 推荐镜头结构

1. 开场钩子：普通球友拍了一段羽毛球视频，不知道动作问题在哪里。
2. 项目出现：ShuttlePoseReview，把手机视频变成动作复盘。
3. 上传流程：选择手机视频，在端上抽帧和识别姿态。
4. 核心结果：视频上叠加人体骨架，识别重发力窗口。
5. 三项指标：击球时机、发力链、回位恢复。
6. 导出功能：下载带骨架的合成 MP4，方便分享给朋友或教练。
7. 缓存功能：复盘过的视频，下次打开直接查看。
8. 开源定位：GitHub 可查看源码、公式和实现，也可以通过国内镜像下载 APK。
9. 收尾：欢迎试用、提 issue、贡献样本和算法建议。

## 可直接交给 HyperFrames 的 Prompt

请生成一个 1080x1920 的竖屏短视频，时长约 70 秒，用来介绍开源项目 ShuttlePoseReview。

视频不需要配音，只需要字幕、动效和画面。字幕使用中文，内容来自 `docs/promo_subtitles_zh.srt`，每次显示 1-2 行，避免遮挡手机界面。字幕风格要适合社交平台：深绿色背景上使用白色粗体字，重点词使用荧光绿色。

画面风格是“羽毛球场 + 开源项目演示 + 竖屏讲解”。背景使用深绿色球场纹理和白色场线。画面左下或右下放一个 GitHub 头像风格的圆形讲解者头像，头像可以轻微点头、眨眼、指向屏幕，但不要生成真实真人脸，不要模仿任何特定创作者。

中间主体展示 ShuttlePoseReview 的手机界面和功能流程：

- 选择或导入视频。
- 正在分析动作。
- 复盘结果页展示骨架姿态。
- 时间线展示多个重发力窗口。
- 三个指标卡片：击球时机、发力链、回位恢复。
- 下载姿态合成视频按钮。

动画要有节奏，但不能遮挡 UI。每个功能点使用简短标注、箭头、框选或扫描线强调。不要使用夸张营销词，不要承诺专业训练效果。重点表达：这是一个开源、端上运行、面向羽毛球爱好者的动作复盘研究工具。

视频最后给出 GitHub 项目名：

ShuttlePoseReview
GitHub: waylean/shuttleposereview

并提示：如果无法访问 GitHub，可以查看 README 中的国内网盘镜像说明。

## 剪辑建议

- 如果你自己配音，建议先录中文口播，再用字幕文件微调时间轴。
- 如果用 GitHub 头像代替真人讲解，建议头像只做轻微动画，不要做过度拟人化口型。
- 如果平台对“AI”字样比较敏感，可以把标题从“AI 动作复盘”改成“姿态识别动作复盘”。
