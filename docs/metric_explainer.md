# ShuttlePoseReview 指标解释

## 一句话版本

ShuttlePoseReview 不是直接识别羽毛球，也不是判断真实球速。它先从视频中提取人体 2D 骨架，再从肩、肘、腕、髋、膝、踝这些点位计算动作窗口、关节角度、速度和恢复状态，最后把明显重发力动作转化为三个复盘指标：

- 击球时机：这一拍是否在更合适的位置和准备状态下完成发力。
- 发力链：这一拍是否呈现出从下肢、躯干、手臂到手腕的连续发力节奏。
- 回位恢复：这一拍打完后，身体是否能较快回到可衔接下一拍的稳定状态。

当前检测的是“明显重发力窗口”，不是全部击球次数。放网、轻挡、过渡球可能真实发生了击球，但不一定会被当成一个重发力窗口单独评分。

## 从骨架到指标

系统使用的核心骨架点位来自人体 2D pose：

| 部位 | 点位 |
|---|---|
| 肩 | 左肩 11，右肩 12 |
| 肘 | 左肘 13，右肘 14 |
| 腕 | 左腕 15，右腕 16 |
| 髋 | 左髋 23，右髋 24 |
| 膝 | 左膝 25，右膝 26 |
| 踝 | 左踝 27，右踝 28 |

系统不会直接把像素速度当作身体能力，而是会先做归一化：

- 手腕速度 = 手腕点位在连续帧中的移动速度。
- 归一化手腕速度 = 手腕速度 / 躯干长度。
- 肘角 = 肩-肘-腕形成的夹角。
- 膝角 = 髋-膝-踝形成的夹角。
- 躯干转动代理 = 肩线与髋线的角度差变化。
- 手腕高度 = 手腕相对肩部的高度，再结合躯干长度理解。

这样做的目的是减少不同视频视野大小、人物远近、分辨率不同造成的波动。

## 第一步：找明显重发力窗口

系统会先计算每一帧的动作强度：

```text
动作强度 ~= 手腕速度 x 手腕高度加成 x 主动手臂可见度修正 - 低位惩罚
```

然后在整段视频中找动作强度的局部峰值。为了避免同一次挥拍被重复计算，相邻重发力窗口之间需要保持约 0.75 秒以上间隔。

在 `IMG_5870` 当前版本中，系统识别到 6 个明显重发力窗口：

```text
第 1 次：0.43s
第 2 次：4.74s
第 3 次：8.30s
第 4 次：12.57s
第 5 次：16.77s
第 6 次：21.18s
```

这些不是全部击球次数，而是适合做发力复盘的动作窗口。

## 指标 1：击球时机

击球时机回答的问题是：这一拍是不是在更合适的位置完成发力？

当前主要看四类信息：

| 信息 | 含义 |
|---|---|
| 手腕高度 | 发力点附近，手腕是否处在更容易完成高点击球的位置 |
| 发力点肘角 | 肘部是否处在较合理的击球伸展区间 |
| 准备期姿态 | 击球前是否有架拍、侧身或身体打开的迹象 |
| 主动手臂可见度 | 这一拍的肘腕数据是否足够可读 |

当前权重：

```text
击球时机 = 46% 手腕高度
        + 24% 发力点肘角
        + 20% 准备期姿态
        + 10% 主动手臂可见度
```

它适合提醒业余球友：击球点是否偏低、是否太仓促、是否没有提前准备。

## 指标 2：发力链

发力链回答的问题是：这一拍是否有从身体到手腕的连续加速过程？

当前不是要求某几个峰值必须严格按顺序出现，因为普通手机视频会受到视角、遮挡、帧率影响。现在采用“时间窗口能量”的方式：

| 时间窗口 | 观察内容 |
|---|---|
| 发力点前 0.65s 到 0.22s | 膝部加载和释放 |
| 发力点前 0.38s 到 0.06s | 躯干打开、肘部加速 |
| 发力点前 0.18s 到发力点后 0.08s | 手腕鞭打速度 |

当前综合：

- 下肢能量。
- 躯干转动能量。
- 肘部角速度。
- 手腕归一化速度。
- 这些能量是否大体从前段转移到后段。
- 击球前是否有膝部加载。

它适合提醒业余球友：是否只是在抡手，还是有下肢、躯干、手臂、手腕的连续节奏。

## 指标 3：回位恢复

回位恢复回答的问题是：这一拍打完以后，身体是否能回到可衔接下一拍的状态？

当前主要看发力点之后约 1.2 秒：

| 信息 | 含义 |
|---|---|
| 恢复时间 | 手腕、肘、膝速度多久降到稳定阈值 |
| 残余动作 | 打完以后手臂和身体是否还在大幅摆动 |
| 姿态回正 | 肩髋角度是否回到更稳定的位置 |

当前权重：

```text
回位恢复 = 54% 恢复时间
        + 28% 残余动作
        + 18% 姿态回正
```

它适合提醒业余球友：是否打完一拍后身体散掉、脚步停住、下一拍准备不足。

## 一个真实例子：IMG_5870 第 5 次重发力

第 5 次重发力发生在约 `16.77s`。

| 指标 | 分数 | 关键数据 |
|---|---:|---|
| 击球时机 | 80 / 100 | 发力点手腕高度约为躯干长度的 0.69 倍，窗口内最高约 0.93 倍 |
| 发力链 | 70 / 100 | 膝部能量 820.5，躯干能量 490.6，肘部能量 3636.5，手腕鞭打 14.24 |
| 回位恢复 | 40 / 100 | 发力后约 0.43s 找到稳定帧，但残余动作和姿态回正仍拖低分数 |

这说明这拍的击球窗口并不差，发力链也有明显能量，但打完之后的身体控制和衔接准备仍然是短板。

对业余球友来说，这类结果比单独说“动作快不快”更有用：它能把问题拆成“打到点没有”“力有没有传上来”“打完能不能接下一拍”。

## 对外表述建议

推荐说法：

> 我们把手机视频里的身体骨架转成动作数据，再从明显重发力窗口里计算击球时机、发力链和回位恢复。它不是裁判，也不是测速仪，而是帮助业余球友看懂自己这一拍哪里更像问题。

不推荐说法：

> AI 已经判断出真实球速、真实发力大小、专业动作标准。

更准确的边界：

- 适合：同一用户、同一机位、同类动作之间做趋势对比。
- 谨慎：不同机位、不同距离、不同人之间直接比较绝对分数。
- 不适合：用单机位 2D pose 声称真实 3D 生物力学或真实羽毛球速度。
